📊 STORAGE DETAILS:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

✅ Database: MongoDB
✅ Collection: flashcardsets
✅ Persistent Storage: Docker volume (flashcard_mongodb_data)
✅ User Association: user field (ObjectId reference)
✅ Data Survives: Container restarts, server reboots

📦 WHAT GETS STORED:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

When a user creates a deck, MongoDB stores:

{
  _id: "unique-deck-id",
  title: "Deck Name",
  fileName: "original-file.pdf",
  user: "user-id-here",           ← OWNER'S USER ID
  cards: [
    {
      question: "...",
      answer: "...",
      section: "...",
      difficulty: "..."
    }
  ],
  csvData: "...",
  createdAt: "2024-01-15T10:30:00Z"
}

🔐 USER ISOLATION:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

✅ Each deck has "user" field
✅ User A's decks: { user: "user-a-id" }
✅ User B's decks: { user: "user-b-id" }
✅ Query filters: FlashcardSet.find({ user: userId })
✅ Result: Each user sees ONLY their decks

📁 PHYSICAL STORAGE:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Location: /var/lib/docker/volumes/flashcard_mongodb_data/_data
Type: MongoDB BSON files
Persistence: ✅ Permanent (until volume deleted)
Backup: ✅ Can be backed up via mongodump

🔄 DATA FLOW:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

CREATE DECK:
User → Frontend → POST /api/flashcards/create-deck
       ↓
Backend: new FlashcardSet({ user: userId })
       ↓
MongoDB: Writes to disk (flashcardsets collection)
       ✅ STORED PERMANENTLY

LOAD DECKS:
User → Frontend → GET /api/flashcards
       ↓
Backend: FlashcardSet.find({ user: userId })
       ↓
MongoDB: Reads from disk
       ✅ LOADED FROM DATABASE
